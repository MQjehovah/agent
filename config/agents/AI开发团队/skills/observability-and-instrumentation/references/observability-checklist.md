# 可观测性检查清单

## 结构化日志
- [ ] 日志输出为 JSON 格式，方便日志系统解析
- [ ] 每个日志条目包含：时间戳、级别、请求 ID、服务名
- [ ] 不同环境（开发/预发/生产）日志级别配置合理
- [ ] 敏感信息（密码、Token）不在日志中输出

## RED 指标
- [ ] 每个接口/端点记录了 **Rate**（请求速率）
- [ ] 每个接口/端点记录了 **Errors**（错误数和错误率）
- [ ] 每个接口/端点记录了 **Duration**（延迟分布，p50/p95/p99）
- [ ] 指标按状态码、路径、方法维度打标

## 健康检查端点
- [ ] 提供 `/health` 或 `/readyz` 端点
- [ ] 健康检查包含关键依赖状态（数据库、缓存、外部服务）
- [ ] 存活检查（liveness）与就绪检查（readiness）分离

## 错误追踪
- [ ] 未捕获异常被全局捕获并上报
- [ ] 错误报告包含堆栈、上下文、请求参数
- [ ] 关键错误有自动告警通知

## 分布式追踪
- [ ] 关键路径（用户请求流）已接入分布式追踪
- [ ] 跨服务请求传递 Trace ID / Span ID
- [ ] 数据库查询、外部调用等子操作记录为子 Span

## 症状告警
- [ ] 告警基于用户可见的症状（错误率升高、延迟飙升）
- [ ] 避免基于 CPU / 内存等基础设施指标的告警
- [ ] 告警有明确的阈值和持续时间
- [ ] 无告警疲劳——每个告警都应 actionable
