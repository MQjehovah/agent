# 无障碍检查清单（WCAG 2.1 AA）

## 键盘导航
- [ ] 所有交互元素可通过 Tab / Shift+Tab 到达
- [ ] 焦点顺序符合视觉顺序（DOM 顺序）
- [ ] 所有功能均可通过键盘独立完成（无鼠标陷阱）
- [ ] Escape 键可关闭弹窗/下拉菜单

## 屏幕阅读器
- [ ] 非文本内容有替代文本（`alt` / `aria-label`）
- [ ] 自定义组件有正确的 ARIA 角色（role）
- [ ] 动态内容更新使用 `aria-live` 区域
- [ ] 标题层级（h1→h6）合理，无跳级

## 颜色对比度
- [ ] 普通文本对比度 ≥ 4.5:1
- [ ] 大号文本（≥18px 或 ≥14px 加粗）对比度 ≥ 3:1
- [ ] 图标和 UI 组件有足够对比度（≥ 3:1）

## 焦点指示器
- [ ] 焦点状态有清晰可见的轮廓（outline）
- [ ] 不要移除默认的 `:focus` 样式而不提供替代
- [ ] `:focus-visible` 用于键盘焦点 vs `:focus` 用于鼠标点击

## 表单标签
- [ ] 每个输入框都有对应的 `<label>` 元素
- [ ] label 通过 `for` 属性与 input 关联，或用 aria-label
- [ ] 必填字段和错误提示已通过文字标明

## 图片替代文本
- [ ] 信息性图片有描述性的 `alt` 文本
- [ ] 装饰性图片使用空 `alt=""` 或 CSS 背景图
- [ ] 复杂图片（图表）提供了长描述或数据表格
