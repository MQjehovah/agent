## 触发条件

当机器人出现以下任一现象时，激活本技能：

1. 机器人与工作站反复对接（反复对桩）
2. 机器人停在工作站前但不对接（不对桩）
3. 机器人停留在清洁区域，未返回工作站，上报"无法返回工作站"或"前往充电点失败"

## 前置知识

- 机器人具备自恢复逻辑，默认尝试对接5次，失败后上报错误
- 对接前置条件：
  - Lora通讯版本需成功预约工作站（ID正确、工作站空闲）
  - 2D激光雷达需能检测到工作站的反光贴
- 导航无法规划路径时，可能触发回站动作，若卡在禁行区内则上报故障

## 诊断流程

### 故障现象1：反复对桩

等待机器人最终报错，根据报错类型排查：

| 报错类型 | 可能原因 | 排查方法 |
|----------|----------|----------|
| 对桩超时 | 极片dock未触发 | 检查金属极片是否氧化（通断测试）；手动对接后检查极片接触（高低差<±2mm）；若仍无触发，更换微动开关版本dock |
| 加水异常 | 清水液位未上升或下降 | 检查水枪出水、液位计浮子是否卡住；高级工程模式单控加水，确认球阀是否开启；检查通讯模式（433/Lora）设置是否正确 |
| 排水异常 | 污水液位未下降 | 清洗污水箱滤网；单控排水，检查排污球阀是否故障 |
| 充电异常 | dock成功但无充电电流 | 确认工作站亮灯；检查极片氧化与接触；测量极片电压、手动充电口电压；重启工作站（GT型号）；检查电池是否故障 |

**操作步骤:**
1. 调用 `get_device_detail(sn)` 获取当前状态和故障列表
2. 查看 `faultDTOList` 中具体的故障码
3. 根据故障码对照上表定位原因
4. 若为硬件问题（极片氧化、液位计卡住等），升级给人工运维

### 故障现象2：不对桩

**操作步骤:**
1. 调用 `get_device_detail(sn)` 确认设备是否到达桩前点（距工作站1.5~2m）
2. 检查 Lora 预约状态：
   - 查看故障码中是否包含工作站预约失败相关信息
   - 若预约失败，检查：工作站是否被占用、Lora模块是否故障、HMI中工作站ID与嗅探ID是否一致
3. 检查 2D 激光雷达是否能识别反光条：
   - 查看 `faultDTOList` 中是否有"对桩超时"相关故障
   - 可能原因：激光雷达高度偏移、工作站安装高度不对、反光贴损坏
4. 检查工作站点位设置是否在 1.5~2m 范围内

### 故障现象3：未返回工作站

**操作步骤:**
1. 调用 `get_device_detail(sn)` 查看设备当前位姿 `position`
2. 调用 `get_cost_map(sn)` 获取感知地图，检查：
   - 是否存在不合理禁行区、虚拟墙
   - 是否存在低于1.1m的门未设置窄门区
   - 是否存在巨大噪点影响路径规划
3. 若设备被困禁行区：
   - 检查设备是否在禁行区内（位置坐标 vs 禁行区范围）
   - 若在禁行区内，需人工移除禁行区或调整任务区域
4. 尝试调用 `relocate(sn, position)` 重定位后，调用 `forward_charge(sn)` 让设备回充

## 通用排查顺序

```
获取设备详情 → 等待最终报错 → 基于报错定位 → 检查dock信号 → 检查加水/排水 → 检查充电极片
```

## 必要工具与日志

- `get_device_detail` — 设备状态和故障列表
- `get_chassis_info` — 底盘话题数据
- `get_clean_info` — 清洁组件数据
- `get_cost_map` — 感知地图

## 常见问题速查

| 现象 | 优先检查 | 常见原因 | 能否远程处理 |
|------|----------|----------|-------------|
| 反复对桩 | dock触发、加水、排水、充电 | 极片氧化、液位计卡住、通讯模式错误 | 部分可（通讯模式） |
| 不对桩 | 预约状态、反光条识别、点位距离 | 工作站被占用、ID不一致、Lora故障 | 部分可（ID配置） |
| 不返回工作站 | 地图设置、禁行区 | 路径被阻断、被困禁行区 | 可（重定位+回充） |
